# 我的每日头条开发计划

## 1. 项目定位

“我的每日头条”是一个面向个人和小团队的信息聚合平台。用户可以自定义信息源，系统负责采集、标准化、去重、搜索、过滤和展示，最终形成用户自己的每日信息看板。

核心目标：

- 支持用户配置自己的信息源，包括标准 API、RSS、RSSHub、网页爬虫、AI Agent 草稿和 Webhook 推送。
- 支持多种信息展示模式，包括列表、宫格、紧凑列表和时间线。
- 支持全文搜索、二次过滤、已读、收藏、隐藏、标签和保存搜索。
- 支持管理员推送公共订阅源，普通用户可选择是否订阅。
- 支持外部 Agent 使用独立 token 认证访问，并按授权范围接出用户数据。
- 支持亮色、暗色、多语言和个人展示偏好。
- 架构低耦合，适合多个 Agent 或多个工程师并行开发。

## 2. 默认技术选型

| 层级 | 默认选型 | 说明 |
| --- | --- | --- |
| 前端 | React、TypeScript、Vite | 单页应用，便于快速开发和独立部署 |
| UI 组件库 | Ant Design、ProComponents | 统一后台管理、表单、表格、卡片、主题和国际化体验 |
| 后端 API | Python、FastAPI | 更适合爬虫、AI Agent、RSS 和任务调度生态 |
| ORM 与迁移 | SQLAlchemy 2、Alembic | SQLite 起步，后续兼容 MySQL |
| 默认数据库 | SQLite | v0.x 单机开发和轻量部署 |
| 生产数据库 | MySQL | v1.0 后支持多用户和更大数据量 |
| 搜索 | Meilisearch | 支持全文搜索、过滤、排序和中文搜索体验 |
| 定时任务 | APScheduler | 支持 interval、cron、固定时间和手动触发 |
| 爬虫 | Scrapling | 作为网页爬虫连接器，不替代 API/RSS/RSSHub |
| RSS 生态 | feedparser、RSSHub | feedparser 解析 RSS/Atom，RSSHub 接入主流站点路由 |
| HTTP 客户端 | httpx | 标准 API、Webhook、RSSHub 请求 |
| 国际化 | react-i18next | 前端多语言，后端错误码可映射翻译 |

## 4. 功能总清单

### 4.1 信息源

- 标准 API 源：
  - 支持 GET/POST。
  - 支持 Header、Query、Body、认证配置。
  - 支持字段映射、分页、游标和预览测试。
- RSS/Atom/JSON Feed 源：
  - 支持手动添加。
  - 支持 OPML 导入和导出。
  - 支持自动识别标题、摘要、正文、图片、作者和发布时间。
- RSSHub 源：
  - 支持配置 RSSHub base URL。
  - 支持用模板生成常见站点路由。
  - 用于接入 Bilibili、新闻站、论坛、社区和公开热榜。
- Scrapling 爬虫源：
  - 支持静态页面抓取。
  - 支持动态页面抓取。
  - 支持 CSS selector、XPath 或 adaptive selector。
  - 支持分页、详情页抓取、预览测试、域名限制、频率限制。
- AI Agent 源：
  - 用户输入 Markdown 需求。
  - Agent 生成信息源草稿、字段映射或爬虫 recipe 草稿。
  - 草稿必须由用户确认后才能启用。
- Webhook/API 推送源：
  - 外部系统可通过 token 推送条目。
  - 适合个人脚本、自动化工具和未来插件系统。

### 4.2 Agent Token 与数据接出

- Token 认证：
  - 用户可创建专用于外部 Agent 的访问 token。
  - token 创建后只展示一次，服务端只保存 hash。
  - 支持名称、过期时间、吊销、轮换、最近使用时间和使用来源记录。
  - 支持按 scope 授权，避免外部 Agent 获得完整账号权限。
- 推荐 scope：
  - `feed:read`：读取已授权信息流。
  - `feed:search`：调用搜索和二次过滤。
  - `sources:read`：读取信息源和订阅元数据。
  - `sources:write`：创建或更新信息源草稿。
  - `exports:create`：创建数据接出任务。
  - `exports:download`：下载导出结果。
  - `agent:draft`：调用 Agent 草稿生成。
- 数据接出：
  - 支持 JSON、NDJSON、CSV、OPML 格式。
  - 支持按时间、来源、标签、收藏状态、搜索条件导出。
  - 支持同步小批量读取和异步大批量导出任务。
  - 支持外部 Agent 拉取 feed、搜索结果、订阅源清单和用户授权的条目状态。
  - 所有接出行为写入审计日志，并受速率限制和最大导出量限制。

### 4.3 信息展示

- 首页模式：
  - 今日信息流。
  - 全部信息。
  - 未读。
  - 收藏。
  - 稍后读。
  - 按来源分组。
  - 按标签分组。
  - 按时间线展示。
- 视图模式：
  - 列表。
  - 宫格。
  - 紧凑列表。
  - 时间线。
- 条目字段：
  - 标题、摘要、正文 Markdown、图片、链接、作者、来源、发布时间、抓取时间、标签。
- 用户操作：
  - 标记已读。
  - 收藏。
  - 隐藏。
  - 添加标签。
  - 打开原文。
  - 复制链接。
  - 反馈重复或低质量内容。

### 4.4 搜索与二次过滤

- 全文搜索范围：
  - 标题。
  - 摘要。
  - 正文。
  - 作者。
  - 来源名称。
  - 标签。
- 过滤条件：
  - 来源。
  - 来源类型。
  - 标签。
  - 时间范围。
  - 语言。
  - 已读/未读。
  - 收藏/未收藏。
  - 是否包含图片。
  - 是否有 AI 摘要。
  - 包含关键词。
  - 排除关键词。
- 搜索增强：
  - 搜索历史。
  - 保存搜索。
  - 默认过滤器。
  - 关键词高亮。
  - 按相关度、发布时间、抓取时间排序。

### 4.5 用户、角色和组

- 管理能力：

  - 角色分配。
  - 组管理。
  - 公共信息源模板管理。
  - 订阅推送。
  - 采集任务审计。

### 4.6 定时抓取

- 调度类型：
  - 手动。
  - 固定间隔。
  - Cron 表达式。
  - 每日固定时间。
- 运行控制：
  - 暂停。
  - 立即抓取。
  - 最大抓取频率。
  - 单次超时。
  - 最大条目数。
  - 失败重试。

- 任务可观测：
  - 最近一次抓取状态。
  - 最近错误。
  - 抓取条目数。
  - 新增条目数。
  - 去重条目数。
  - 平均耗时。

## 5. 版本详细计划

### 5.1 v0.1 基础骨架

目标：

- 建立可运行的前后端基础工程。
- 建立认证、数据库迁移、统一 API 返回格式、前端基础布局。

功能范围：

- Monorepo 目录结构。
- FastAPI 应用启动。
- SQLAlchemy 和 Alembic 初始化。
- React、Vite、Ant Design 初始化。
- 登录页、主布局、空白首页。
- JWT 登录和当前用户接口。
- 初始管理员账号创建脚本。

不做什么：

- 不实现真实信息源采集。
- 不接入搜索引擎。
- 不实现 AI 和爬虫。

验收标准：

- 开发者能一条命令启动后端和前端。
- 管理员能登录。
- 登录后能进入应用主布局。
- Alembic 能创建 SQLite 表。
- API 文档可访问。

测试要求：

- 后端健康检查测试。
- 登录成功和失败测试。
- 前端登录页渲染测试。

可并行开发模块：

- 后端基础工程 Agent。
- 前端基础 UI Agent。
- 数据库迁移 Agent。

### 5.9 v0.9 Agent Token 与数据接出

目标：

- 让外部 Agent、个人脚本和自动化系统可以用独立 token 访问平台数据，并按 scope 安全接出数据。

功能范围：

- Agent token 创建、查看、吊销、轮换。
- token hash 存储和一次性明文展示。
- scope 授权。
- token 认证入口。
- Agent principal 上下文。
- 数据接出 API。
- 同步小批量数据读取。
- 异步导出任务。
- JSON、NDJSON、CSV、OPML 导出格式。
- 导出审计、速率限制、最大导出量。

不做什么：

- 不让 Agent token 等同于用户完整登录态。
- 不提供无限制全库导出。
- 不在 v0.9 实现完整 OAuth2 第三方应用授权。
- 不让 token 跳过用户订阅、组权限和管理员策略。

验收标准：

- 用户能创建一个只读 token，并在创建后看到一次明文 token。
- 外部 Agent 能使用 `Authorization: Bearer <token>` 调用授权 API。
- 未授权 scope 的 API 返回明确权限错误。
- token 被吊销后立即失效。
- Agent 能按搜索条件导出 JSON 或 CSV 数据。
- 大批量导出以任务形式执行，并可下载结果。
- 所有 token 使用和数据接出写入审计日志。

测试要求：

- token 创建、hash 校验、吊销、过期测试。
- scope 权限测试。
- 数据导出格式测试。
- 大批量导出任务测试。
- 审计日志测试。
- 速率限制测试。

可并行开发模块：

- Agent Access Agent。
- Data Export Agent。
- Token 管理 UI Agent。

## 6. 多 Agent 并行开发计划

| Agent | 负责范围 | 主要产物 | 依赖 |
| --- | --- | --- | --- |
| Backend Foundation Agent | FastAPI、配置、数据库、迁移、统一响应 | API skeleton、Alembic、settings | 无 |
| Frontend Foundation Agent | React、Ant Design、路由、布局、请求封装 | App shell、主题基础、API client | 无 |
| Auth RBAC Agent | 登录、用户、角色、组、权限 | auth API、RBAC service、测试 | Backend Foundation |
| Source Agent | 信息源、模板、订阅 | sources API、schema、测试 | Auth RBAC |
| Connector Agent | API、RSS、RSSHub 连接器 | connector interface、实现、fixture | Source |
| Ingestion Agent | 抓取、标准化、去重、条目写入 | ingestion service、job logs | Connector |
| Feed UI Agent | 首页、列表、宫格、条目操作 | feed pages、components | Frontend Foundation、Feed API |
| Search Agent | Meilisearch、索引、搜索 API | indexer、search API、测试 | Ingestion |
| Scheduler Agent | APScheduler、调度、任务状态 | schedule service、worker | Source、Ingestion |
| Admin Agent | 用户管理、模板、订阅推送 | admin pages、admin API | Auth RBAC、Source |
| Crawler Agent | Scrapling recipe、预览、限制 | crawler connector、recipe UI | Connector、Source |
| AI Agent | 模型配置、草稿生成、工具接口 | agent runtime、draft UI | Source、Connector |
| Agent Access Agent | Agent token、scope、认证上下文 | token API、scope 校验、审计 | Auth RBAC、Feed API |
| Data Export Agent | 数据接出、导出任务、格式转换 | export API、JSON/CSV/NDJSON/OPML | Agent Access、Feed/Search |
| I18n Theme Agent | 多语言、主题、偏好 | i18n resources、theme switcher | Frontend Foundation |
| QA Agent | 单测、API 测试、E2E、测试数据 | test suite、mock data | 各模块稳定接口 |

并行规则：

- 先冻结 OpenAPI 草案、数据库 migration 草案和事件契约。
- 每个 Agent 只能修改自己负责的模块，跨模块变更必须先更新接口文档。
- 每个模块必须提供 mock 数据和最小验收测试。
- 后端模块之间通过 service interface 调用，不直接跨模块访问内部 repository。
- 前端模块之间通过页面路由、共享 API client 和公共 UI primitives 协作。

## 8. 验收总标准

v1.0 之前必须满足：

- 普通用户能创建 RSS、API、RSSHub 信息源并看到聚合条目。
- 普通用户能搜索、过滤、收藏、隐藏和标记已读。
- 管理员能管理用户、角色、组和公共模板。
- 管理员能推送订阅源，普通用户能接受或忽略。
- 信息源能按计划自动抓取。
- Scrapling 爬虫源有预览、域名限制、频率限制和任务日志。
- AI Agent 只能生成草稿，不能绕过确认直接启用信息源。
- 外部 Agent 能通过 scoped token 读取和导出授权范围内的数据。
- 亮色、暗色、中文、英文完整覆盖核心页面。
- SQLite 和 MySQL 均能通过核心测试。

## 9. 风险和约束

- 第三方平台接口可能变化，必须通过模板和 connector 隔离变化。
- 爬虫存在合规和稳定性风险，默认只支持公开、授权、低风险来源。
- AI 生成的配置可能错误，必须经过预览和用户确认。
- Agent token 一旦泄露会造成数据外流，必须支持过期、吊销、scope、速率限制和审计。
- Meilisearch 是搜索引擎，不是主数据库，不能把核心业务状态只存在搜索索引。
- SQLite 适合 v0.x 和轻量部署，多用户生产场景应切换 MySQL。
- 多 Agent 并行开发需要接口先行，否则容易出现重复实现和跨模块耦合。
