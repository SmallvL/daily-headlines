设计要点

  • 布局：卡片水平垂直居中，四周留白。结构改动最小，与现有 LoginPage 兼容。
  • 卡片：大圆角（20px）、柔和弥散阴影、纯白背景，营造悬浮感。
  • 品牌：渐变圆角 Logo 图标 + 应用名 + 一句话 slogan，强化品牌记忆。
  • 表单：左侧对齐的输入框，圆角输入框，主色登录按钮，错误提示在卡片内顶部。
  • 动效：卡片轻微上浮入场（Framer Motion），输入框聚焦时边框颜色过渡，按钮悬停微缩放。
  • 主题：登录页背景使用浅灰渐变，不跟随暗色主题；进入应用后由设置控制主题。
  • 响应式：移动端卡片宽度自适应（min-width + 最大宽度），内边距适当缩小。

技术方案

  • 新增依赖：framer-motion（入场动效）
  • 重构文件：modules/auth/LoginPage.tsx
  • 样式策略：登录页专用样式集中在组件内（styled/inline），不污染全局 styles.css
  • 保留：Ant Design Form/Input/Button 与现有登录 API 逻辑

Pros

  • 改动最小，与现有路由/状态完美兼容
  • 与后续混合式导航的简洁风格协调
  • 实现快速，风险低

Cons

  • 品牌展示面积比分屏方案小
  • 大面积留白在超大屏上可能显得空旷

确认这套方案后，我会写入正式设计文档，再生成实现计划。
